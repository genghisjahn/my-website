Small Files, Swift Pages: The Case for Hand-Built Sites

By Author Name

Speed favors the humble page. A single HTML file, a whisper of CSS, and a browser does the rest. The result reads well on a phone and wastes no battery.

Each story owns its style. No shared bundle. No fragile theme. You compile tag lists and monthly archives at build time so every link lands on a static file.

“Type first. Effects later. If it hurts the reading, it isn’t design.”

Typography does the heavy lifting. Keep the line length near 60–70 characters. Use generous line height. Let hyphenation help justification on wider screens.

A page like this degrades gracefully. Old browsers ignore niceties like columns and small caps. The copy still reads cleanly in one column on mobile.
